1.命名约定
	1.1包
		包名全部小写字母，本系统中包组织规则如下：
		com.zdfd.elec.action.模块名	------服务端控制层代码
		com.zdfd.elec.模块名.dao		------持久层数据访问代码
		com.zdfd.elec.模块名.entity	------持久层实体类
		com.zdfd.elec.模块名.service	------业务实现相关代码
		com.zdfd.elec.util			------与业务无关的辅助代码
		
	1.2类
		类名必须是一个名词，简洁而富于描述，避免缩写（除Url,Html等被广泛使用的缩写）
		类名采用大小写混合方式，每个单词首字母大写，如：Hello, HelloWorld
		
	1.3接口
		接口命名与类相同，接口实现为：接口名+Impl，接口与实现放在同一包下
		
	1.4方法
		方法名是一个动名词词组，要求具有语义
		采用大小写混合形式，首字母小写，其后单词首字母大写，不要用缩写，第一个单词一般为动词
		方法名中不得使用特殊字符，如:$,&,_,-等
		
	1.5变量
		常量所有字母大写，每个单词之间用下划线隔开，如：PLAN_STATE
		除常量外，其他变量采用大小写混合格式，变量第一个字母小写，其后每个单词首字母大写
		变量名尽量简短且有意义，易于记忆，若是集合变量应使用复数形式
		变量中不得使用特殊字符，如:$,&,-,_等
		
2.注释
	2.1类
		每个类开头必须使用文档注释格式，注释内容包括创建者、用途，格式如下：
		/**
		 * 此类的用途
		 * @author 创建人
		 */
		public class ClassName {
		}
		
	2.2方法
		方法注释使用文档注释格式，注释内容包括用途、参数、返回值，格式如下：
		/**
		 * 用途描述
		 * @param s1
		 *		s1说明
		 * @param s2
		 *		s2说明
		 * @return
		 * @throws Exception
		 */
		 public String test(String s1, String s2) throws Exception {
		 }
		 
		 方法内根据需要可以采用单行注释和多行注视，注释格式如下：
		 // 单行注释
		 
		 /*
		  * 多行注释
		  */
		 
	2.3变量
		com.zdfd.elec.模块名.entity包下的实体类变量应采用文档注释格式，格式如下：
		/**
		 * 编号
		 */
		private Integer id;
